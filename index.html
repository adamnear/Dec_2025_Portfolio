Project Summary: Wind Turbine Data Analysis

Building an ETL Pipeline for Geospatial Data Analysis: Nova Scotian Turbine Report
This project involved developing a robust ETL (Extract, Transform, Load) workflow to analyze the relationships between wind turbine technical specifications. Utilizing R and the tidyverse, I processed raw geospatial data to examine correlations between Turbine Rated Capacity, Hub Height, and Rotor Diameter.

Key Technical Workflows:

  • Environment & ETL Setup: Configured a data pipeline using GGally and patchwork libraries. Implemented data type enforcement using the mutate function to ensure high-precision numeric analysis.
  • Numeric Variable Analysis: Generated comprehensive summary statistics (mean, median, SD, IQR) to establish a baseline for the Nova Scotia turbine dataset.
  • Distribution Visualization: Engineered a multi-plot dashboard featuring Density Plots, Histograms, and Box Plots to identify data spikes, skewness, and significant outliers in turbine dimensions.

  • Frequency modelling: Conducted frequency and proportion counts to identify industry standards within the region, identifying the 80m hub height and 2300kW capacity as the most frequent regional configurations.

Conclusion: This project demonstrates proficiency in statistical programming and geospatial data cleaning. Project Summary: The Rivers of South America

Domain: Land Analysis Skill: GIS / Cartography

The Rivers of South America is a high-resolution cartographic study that visualizes the continent's complex water networks. By processing HydroSHEDS global data, the project shows the relationship between geographic scale and water volume, specifically focusing on the massive physical presence of the Amazon drainage system. The map uses Strahler Stream Order attributes—a system that ranks rivers by their number of tributaries—to drive a visual hierarchy, ensuring the line thickness and color accurately reflect the size and flow of each river system.

Key Technical Workflows:

  • Data Processing: Sourced and clipped HydroSHEDS shapefile data specifically to the South American continental boundary to isolate the primary drainage basins.
  • Dynamic Symbology: Utilized the Expression Builder in QGIS to define line widths based on the Strahler Order ("ORD_STRA" / 10), creating a visual hierarchy that reflects physical river volume.
  • Stream Order Classification: Applied categorized symbology to differentiate stream orders, utilizing a graduated color ramp from deep blue to white to represent the transition from major river trunks to high-altitude headwaters.
  • Professional Layout Design: Developed a high-contrast print layout with a black canvas, disabling background rendering to allow the hydrological network to stand out as the primary focal point.

Outcomes: The resulting map effectively highlights the Amazon basin’s intricate structure, demonstrating advanced competency in vector data management and thematic cartography. This workflow can be adapted for any global region to support climatology, hydrology, or environmental planning studies.

Project Summary: Supervised Land Classification

Domain: Land Analysis / Forestry Skill: Remote Sensing / GIS

This project involved performing a complex Supervised Image Classification of Landsat 9 multispectral imagery covering the Annapolis Valley, Nova Scotia. Using Catalyst Focus and ArcGIS Pro, I transformed raw spectral data into a refined thematic map identifying six primary land cover information classes.

Key Technical Workflows:

  • Image Preparation & Pan-sharpening: Processed Level 1 Landsat 9 bundles, performing pan-sharpening and clipping to a specific area of interest (AOI) to enhance spatial resolution for feature identification.
  • Supervised Training & Signature Analysis: Manually digitized training areas to create distinct spectral classes. Evaluated signature quality using Bhattacharrya Distance for separability, scatter plots for spectral overlap, and histograms for class homogeneity.
  • Class Aggregation & Post-Processing: Aggregated various spectral classes (e.g., bare soil, rock, and quarry) into higher-level Information Classes (e.g., Barren). Applied a Sieve Filter to reduce "salt-and-pepper" noise while maintaining critical linear features like transportation networks.
  • Rigorous Accuracy Assessment: Conducted a non-biased accuracy assessment using 25+ independent random reference samples per class. Generated Confusion Matrices and Kappa Coefficients to statistically validate the classification reliability.

Outcomes: The final output was a 24” x 36” professional-grade map following NRCAN standards. The project demonstrated proficiency in the full remote sensing lifecycle—from raw sensor data ingestion and statistical validation to final cartographic production.

Project Summary: Best Path Raster Analysis

Domain: Land Analysis Skill: Raster Analysis / Modelling

Best Path Raster Analysis is a geospatial modelling project focused on the principles of Map Algebra and Raster Analysis using ArcGIS Pro. This project explores the mathematical frameworks used to determine the "path of least resistance" across complex geographic surfaces. By leveraging Cost Functions and PathDistance algorithms, the project demonstrates how to model optimal routes—such as road corridors—by balancing engineering constraints, environmental impacts, and social costs.

Technical & Analytical Contributions:

  • Map Algebra Implementation: Applied core raster modelling concepts, including CostDistance and CostBackLink, to transform geographic units into weighted cost units based on relative and absolute constraints.
  • Corridor modelling Strategy: Developed a multi-criteria impedance grid for road construction, accounting for geotechnical factors (slope, soils, hydrography) and absolute barriers like national parks or water bodies.
  • Advanced Path Analysis: Utilized the PathDistance function to incorporate 3D surface distance, vertical factors, and horizontal resistance, ensuring the model accounted for real-world topographical energy expenditures.
  • Python & Scripting Integration: Conducted the analysis through the ArcGIS Pro Python Window, managing environment settings such as workspace extents, cell sizes, and scratch workspaces to ensure data integrity across iterative runs.
  • Iterative Optimization: Generated directional "Back Link" grids to reconstruct the single most efficient route between designated start and stop source cells, validating the theoretical "least-cost path" against spatial variables.

Outcomes: The project resulted in a robust methodological workflow for automated route selection. By successfully integrating Spatial Analyst global functions with Python scripting, the analysis showcased the ability to translate qualitative site constraints into a quantitative, data-driven spatial model for infrastructure planning and environmental mitigation.
